9 · Data Architecture & Freshness (v2 — Live Stocks Enabled)

Price sources:

  • Crypto (BTC, ETH, HYPE, VVV): CoinGecko public API hit client-side from your browser every 60 seconds. No API key. CORS-enabled. 🟢 dot = live.
  • Stocks (NVDA, PLTR, OKLO, RCAT, TSLA, TSM, ASML): Yahoo Finance blocks browser fetches (no CORS), so live stock prices are served through a tiny Cloudflare Worker you deploy. Setup guide: STOCK_WORKER_SETUP.md. Once the Stock Worker URL is set in Settings, the dashboard polls it every 60s and the 🟢 dot lights up. If no worker URL is set, the dashboard falls back to server-baked prices from the last scheduled run (5am / 11am / 5pm / 11pm ET).
  • Manual refresh: Click ⟳ Crypto or ⟳ Stocks buttons in the freshness bar to force an update.

What was fixed: prior dashboard pulled prices via web search, which surfaced stale/synthesized results (HYPE showed $43.27 vs actual $41.19, VVV showed $8.50 vs actual $9.18). That entire code path is retired. Crypto now hits CoinGecko directly; stocks hit Yahoo via your Worker proxy.

Why a Worker and not direct-to-Yahoo: browsers refuse to read cross-origin responses unless the server sends Access-Control-Allow-Origin — Yahoo doesn't. A Cloudflare Worker sits server-side, fetches Yahoo freely, then serves your dashboard with CORS open. Free tier covers ~100k requests/day; you'll use about 10k/day even polling continuously.
